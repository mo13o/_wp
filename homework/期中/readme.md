#### 1. 資料儲存與狀態模擬 (模擬資料庫)

因為我們沒有後端伺服器，所以所有的「會員資料」、「留言」與「購買紀錄」都存放在瀏覽器的緩存中。

* **`localStorage.setItem(key, value)`**
* **具體作用：** 將資料存入瀏覽器。
* **應用場景：** 儲存註冊的帳號密碼 (`shop_users`)、儲存當前登入狀態 (`currentUser`)、記錄最後領獎日期、儲存顧客留言 (`shop_reviews`)。


* **`localStorage.getItem(key)`**
* **具體作用：** 從瀏覽器讀取之前存入的資料。
* **應用場景：** 每次重整網頁時，檢查有沒有 `currentUser` 來決定要不要顯示登出按鈕；讀取商品先前的評價來渲染畫面。


* **`localStorage.removeItem(key)`**
* **具體作用：** 刪除特定紀錄。
* **應用場景：** 實作「登出」功能，清除當前的登入狀態。


* **`JSON.stringify()` 與 `JSON.parse()**`
* **具體作用：** 文字與 JavaScript 物件/陣列的互相轉換。(`localStorage` 只能存純文字)。
* **應用場景：** 我們將多筆評價陣列轉成文字存入（`stringify`），讀取時再轉回陣列來跑迴圈（`parse`）。



#### 2. 畫面渲染與操作 (DOM 操作)

控制網頁畫面的顯示、隱藏以及動態產生內容。

* **`document.getElementById(id)`**
* **具體作用：** 鎖定網頁中的特定元素。
* **應用場景：** 抓取輸入框的值、找到要放入商品列表的 `div` 容器。


* **`element.innerHTML`**
* **具體作用：** 動態替換 HTML 標籤內的內容。
* **應用場景：** 根據是否登入，替換右上角的「登入/註冊」按鈕為「👋 某某某，登出」；將留言陣列組裝成 HTML 字串後一次塞入評價區塊。


* **`document.createElement()` 與 `element.appendChild()**`
* **具體作用：** 用程式碼無中生有一個 HTML 標籤，並把它塞進網頁中。
* **應用場景：** 在 `renderProducts` 中，我們針對每一個商品動態創造 `<div class="product-card">`，再把它掛載到畫面上。


* **`element.style.display`**
* **具體作用：** 修改 CSS 的 display 屬性。
* **應用場景：** 控制所有彈出視窗（Modal）。設為 `'flex'` 時顯示視窗，設為 `'none'` 時隱藏視窗。


* **`element.classList.add()` / `.remove()**`
* **具體作用：** 動態新增或移除 CSS Class。
* **應用場景：** 實作「星星評分」功能，點擊時為星星加上 `.active` 來變成亮黃色。



#### 3. 陣列處理與邏輯運算 (資料處理)

處理我們的商品清單、留言清單與購買紀錄。

* **`Array.filter()`**
* **具體作用：** 篩選出符合條件的資料，組成新陣列。
* **應用場景：** 實作「搜尋功能」，比對商品名稱或描述有沒有包含關鍵字。


* **`Array.map()`**
* **具體作用：** 將陣列中的每一筆資料轉換成另一種格式。
* **應用場景：** 在 `loadReviews` 裡，將留言物件的陣列，直接轉換成一段段的 HTML 字串以便顯示。


* **`Array.includes()`**
* **具體作用：** 檢查陣列中是否包含某個特定值。
* **應用場景：** 檢查使用者的「已購買清單」中有沒有當前商品的 ID，藉此判斷要顯示「購買按鈕」還是「留言表單」。


* **`Array.unshift()`**
* **具體作用：** 將新資料塞入陣列的「最前面」。
* **應用場景：** 送出新評價時，確保最新的留言會顯示在最上方。



#### 4. 事件監聽與互動 (觸發時機)

告訴程式什麼時候該執行上述的那些動作。

* **`onclick` (HTML 屬性 / JS 事件)**
* **具體作用：** 監聽滑鼠點擊。
* **應用場景：** 點擊商品打開預覽、點擊按鈕送出表單、點擊背景關閉視窗 (`window.onclick`)。


* **`onkeyup` (HTML 屬性)**
* **具體作用：** 監聽鍵盤按鍵放開的瞬間。
* **應用場景：** 搜尋列的即時搜尋，使用者每打一個字，畫面就立刻刷新結果。


* **`window.onload`**
* **具體作用：** 網頁資源全部載入完成後觸發。
* **應用場景：** 一進網頁就立刻執行「渲染全部商品」與「檢查登入狀態」。



#### 5. 字串與日期工具

* **`String.toLowerCase()`**
* **具體作用：** 把英文字母全轉為小寫。
* **應用場景：** 讓搜尋功能「不分大小寫」都能搜到東西（把關鍵字和商品名都轉小寫後再比對）。


* **`String.trim()`**
* **具體作用：** 刪除字串頭尾的空白。
* **應用場景：** 防止使用者在註冊或登入時，不小心輸入了多餘的空白鍵導致驗證失敗。


* **`Date.toLocaleDateString()`**
* **具體作用：** 取得當地的日期字串（如：`2023/10/25`）。
* **應用場景：** 判斷「每日領獎」是否今天已經領過；作為評價送出時的「留言日期」。

### 💡 指令速查表 (Quick Reference)

| 動作目標 | 使用的 JavaScript 語法 / 指令 |
| --- | --- |
| **讀/寫暫存資料庫** | `localStorage.getItem()` / `setItem()` |
| **字串與物件互轉** | `JSON.stringify()` / `JSON.parse()` |
| **抓取網頁元素** | `document.getElementById('id')` |

| **顯示/隱藏彈出視窗** | `element.style.display = 'flex' 或 'none'` |
| **無縫替換網頁內容** | `element.innerHTML = '<p>新內容</p>'` |
| **控制 CSS 樣式類別** | `element.classList.add('className')` |
| **取得今天日期** | `new Date().toLocaleDateString()` |
